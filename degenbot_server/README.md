# DegenBot

AI-powered crypto trading bot — Telegram interface, Dart backend, multi-chain.

---

## Architecture

├── degenbot_server/             ← Serverpod Mini backend (runs in Docker)
│   ├── bin/main.dart            ← Server entrypoint
│   └── lib/src/
│       ├── config/env.dart      ← Type-safe env vars (envied)
│       ├── models/*.spy.yaml    ← Serverpod model definitions (source of truth)

│       ├── services/
│       │   ├── dto/             ← Serverpod model ↔ Supabase row translators
│       │   ├── repository/      ← All DB reads/writes (speaks Serverpod models)
│       │   ├── dex/             ← DexScreener API client
│       │   ├── intelligence/    ← Multi-source token intelligence pipeline (Step 2)
│       │   │   ├── token_intelligence_report.dart   ← unified data model
│       │   │   ├── goplus_service.dart               ← Layer 2: contract safety
│       │   │   ├── rugcheck_service.dart              ← Layer 2: Solana-specific
│       │   │   ├── tokensniffer_service.dart          ← Layer 2: second opinion
│       │   │   ├── chaingpt_service.dart              ← Layer 4: sentiment
│       │   │   ├── onchain_forensics_service.dart     ← Layer 5: wallet forensics
│       │   │   └── token_intelligence_pipeline.dart   ← orchestrator + AI scoring
│       │   ├── chain/           ← Blockchain execution (on_chain) [Step 4]
│       │   └── risk/            ← Risk calculation [Step 3]
│       ├── endpoints/           ← Serverpod Mini HTTP endpoints
│       └── bot/
│           ├── telegram_bot.dart
│           ├── middleware/      ← Runs before every message
│           ├── handlers/        ← Command + AI handlers
│           └── utils/           ← Message formatters
├── degenbot_client/             ← Generated client (used by Flutter app later)
├── degenbot_flutter/            ← Mobile app (Step 7 — not started yet)

├── docker-compose.yml

## Step-by-step setup

### Step 1 — Prerequisites

```bash
# Install Dart SDK (3.5+)
# https://dart.dev/get-dart

# Install Docker Desktop
# https://docs.docker.com/get-docker/

# Install Serverpod CLI
dart pub global activate serverpod_cli

# Verify
dart --version       # should be >= 3.5.0
docker --version
serverpod version
```

### Step 2 — Clone and install dependencies

```bash
git clone <your-repo-url> degenbot
cd degenbot

# Install all workspace packages in one command
dart pub get
```

### Step 3 — Create your Supabase project

1. Go to [supabase.com](https://supabase.com) → New project
2. Choose a region close to your users
3. Once created: **Project Settings → API**
   - Copy `Project URL` → this is your `SUPABASE_URL`
   - Copy `service_role` key (NOT the `anon` key) → `SUPABASE_SERVICE_ROLE_KEY`
4. Go to **SQL Editor → New Query**
5. Paste the contents of `docs/supabase_schema.sql` and click **Run**

### Step 4 — Create your Telegram bot

1. Open Telegram → search for `@BotFather`
2. Send `/newbot`
3. Follow prompts → you get a token like `123456789:ABCdef...`
4. Copy that token → `TELEGRAM_BOT_TOKEN`

### Step 5 — Get an AI API key

**Option A: Google Gemini (recommended — generous free tier)**
1. Go to [aistudio.google.com](https://aistudio.google.com)
2. Create API key → copy it → `GEMINI_API_KEY`

**Option B: OpenAI**
1. Go to [platform.openai.com](https://platform.openai.com) → API keys
2. Create key → `OPENAI_API_KEY`
3. Set `AI_PROVIDER=openai` in your `.env`

### Step 6 — Configure environment

```bash
cd degenbot_server
cp .env.example .env
# Edit .env and fill in all values
```

### Step 7 — Generate code

```bash
cd degenbot_server

# Generate env.g.dart (from envied annotations)
dart run build_runner build --delete-conflicting-outputs

# Generate Serverpod protocol (endpoints + client)
dart run serverpod generate
```

### Step 8 — Run locally (without Docker)

```bash
cd degenbot_server
dart run bin/main.dart
```

You should see:
```
[INFO] Main: Supabase client ready
[INFO] Main: Telegram bot ready (polling mode)
[INFO] Main: Serverpod Mini running on port 8080
```

Send your bot a `/start` on Telegram — it should respond.

### Step 9 — Run with Docker (recommended)

```bash
# From project root
docker compose up --build

# Check it's healthy
docker compose ps
# Should show: degenbot   running (healthy)
```

## Deployment (Production)

### Option A: Fly.io (easiest, free tier available)

```bash
# Install flyctl
curl -L https://fly.io/install.sh | sh

# Login
fly auth login

# From project root
fly launch   # follow prompts, say YES to Dockerfile

# Set secrets (never commit these)
fly secrets set \
  TELEGRAM_BOT_TOKEN="your-token" \
  SUPABASE_URL="your-url" \
  SUPABASE_SERVICE_ROLE_KEY="your-key" \
  GEMINI_API_KEY="your-key" \
  WEBHOOK_BASE_URL="https://your-app.fly.dev"

# Deploy
fly deploy
```

### Option B: Railway

```bash
# railway.app → New project → Deploy from GitHub
# Add environment variables in the Railway dashboard
# Railway auto-detects Dockerfile
```

### Option C: Any VPS (DigitalOcean, Hetzner, etc.)

```bash
# SSH into your VPS
# Install Docker
curl -fsSL https://get.docker.com | sh

# Clone repo
git clone <your-repo>
cd degenbot

# Create .env with production values
cp degenbot_server/.env.example degenbot_server/.env
# Edit .env

# Start
docker compose up -d

# View logs
docker compose logs -f
```

### Webhook vs Polling

- **Polling** (default when `WEBHOOK_BASE_URL` is empty): Bot asks Telegram for new messages every few seconds. Works anywhere, no public URL needed. Fine for development and low-traffic bots.
- **Webhook** (recommended for production): Telegram pushes messages to your server instantly. Faster, more efficient. Requires a public HTTPS URL. Set `WEBHOOK_BASE_URL=https://your-domain.com`.

## API costs and toggling features on/off

Every intelligence data source can be switched on or off independently —
either by editing `.env` before first run, or live via the `/features`
Telegram command (no restart needed for the live toggle).

| Source | Layer | Cost | Default |
|---|---|---|---|
| DexScreener | 1 — Market data | Free, no key | ON |
| GoPlus Security | 2 — Safety | Free, 30 calls/min | ON |
| RugCheck.xyz | 2 — Safety (Solana) | Free | ON |
| TokenSniffer | 2 — Safety (2nd opinion) | $99/mo for API access | OFF |
| ChainGPT InfoFI | 4 — Sentiment | Credit-based / stake-for-free | OFF |
| Etherscan / BscScan | 5 — On-chain forensics | Free tier | ON |
| Gemini / OpenAI | AI scoring | Free tier generous; ~$0.10–$4/M tokens paid | ON |

**Recommended starting point:** run entirely on the free sources first.
Everything marked ON above costs nothing at normal scan volumes. Add
TokenSniffer and ChainGPT later only once you've decided the extra
signal is worth paying for — the pipeline degrades gracefully without
them (see `token_intelligence_pipeline.dart`, which checks each flag
before calling that service).

To toggle live: send `/features` to your bot. Each data source shows as
a tappable button — green means on, gray means off. Tapping flips it
immediately and the very next token scan picks up the change.

To change what a *fresh* database starts with, edit the `ENABLE_*`
variables in `.env` before first run — see `.env.example` for the full
list with comments on which ones are free vs paid.

## Adding a new command

1. Add the handler method in `lib/src/bot/handlers/command_handlers.dart`
2. Register it in `register()`: `_bot.command('mycommand', _onMyCommand);`
3. If it needs DB access, add the method to the relevant repository
4. If it needs a new endpoint (for Flutter app access), add to the endpoints/ folder and re-run `dart run serverpod generate`

## Adding a new AI tool

1. Add a new `Tool(...)` entry to the `_buildTools()` list in `ai_handler.dart`
2. Write a clear `description` — the LLM reads this to decide when to use it
3. The `execute` function receives `Map<String, dynamic> args` and returns `String`
